创意介绍

想解决什么问题

在信息爆炸的时代，知识工作者面临三大核心困境：知识碎片化——笔记散落在各处，难以形成体系；思维线性化——传统文档无法展现知识点之间的复杂关联；创作割裂化——记录想法、整理资料、深度编辑需要在多个工具间切换。

产品形态

AstroKnot 是一款基于 Electron + Three.js 的桌面端知识管理应用。它将每个知识点抽象为 3D 空间中的「星节点」，节点间的关联以螺旋连线呈现，配合星云粒子效果，让整个知识网络如宇宙星图般直观可见。用户可以在 3D 视图中自由探索、在 2D 思维导图中整理结构，双击任意节点即可打开功能完整的富文本编辑器进行深度创作，或切换到代码沙盒 IDE 运行前端项目。内置版本控制系统记录每一次变更，终端、AI 对话、日历排班等系统级工具一应俱全。

核心创新点

  • 三维空间知识映射 — 打破传统二维笔记的局限，用 3D 星图形式展现知识的立体结构
  • 全栈式创作环境 — 集成富文本、表格、公式、代码、音频、视频等 12+ 种内容类型于单一节点内
  • 节点内代码沙盒 — 每个节点可嵌入 VSCode 风格 IDE，支持 HTML/CSS/JS 实时预览、终端、esbuild 打包
  • AI 深度融合 — 内置 Chat/Agent 双模式 AI 对话，支持项目上下文感知的工具调用
  • 版本控制系统 — 类 Git 的版本图，支持提交、分支、diff、回滚，每个节点都有完整的变更历史
  • 视图自由切换 — 3D 星图 / 2D 思维导图无缝切换，适应不同思考场景
  • 便携式数据架构 — 所有用户数据（项目、笔记、设置）集中在应用目录下，整个文件夹移动即可无损迁移

目标用户及痛点

学生 / 研究者

使用场景：课程笔记整理、论文文献管理、课题研究梳理

当前痛点：笔记分散在不同 App 中，复习时难以建立跨学科的知识关联；文献和笔记之间缺少直观的链接关系。

知识工作者

使用场景：产品需求文档、技术方案沉淀、行业研究分析

当前痛点：传统文档是线性的，无法体现业务模块之间的依赖关系；需要同时使用 Notion（记录）+ XMind（梳理）+ Word（输出），效率低下。

开发者 / 技术写作者

使用场景：技术博客写作、开源项目文档、API 设计文档

当前痛点：需要频繁插入代码块、数学公式、架构图，现有工具对混合内容的支持不够友好；缺乏能同时承载文字、代码、图表的创作环境。

创意工作者

使用场景：剧本大纲构思、游戏世界观设定、创意灵感捕捉

当前痛点：想法是网状的，但工具是线性的；需要一种能像「思维导图」一样发散、又像「文档」一样深入的表达方式。
